References that
survive renames.

DataIndexer is a game data management plugin for UE5. Every row is identified by a GUID (FDataIndexerPrimaryKey), so references survive renames, moves, and reimports. A type-safe C++ / Blueprint API and a centralized Schema keep maintenance costs low as your data grows.

01 / メンテナンス性

リネームしても、参照は壊れない。

DataTable の FName 参照はリネームで壊れます。DataIndexer は GUID（FDataIndexerPrimaryKey）で識別するため、名前が変わっても参照は生き続けます。

  • GUID ベースの PrimaryKey

    キー名を変更しても参照はそのまま維持。Blueprint やコードの参照が壊れません。

  • Key 参照の追跡

    どのアセット・コードから参照されているかをエディタ上で確認できます。

02 / コンポジション

継承と統合で、データを重ねる。

  • Repository の継承と統合

    UCompositeDataTable が担っていた「親テーブルの統合」を継承。複数の Repository をマージする機能も標準サポートです。

  • 継承時の上書き

    子 Repository 内で特定の Row だけを上書き。UCompositeDataTable にはなかった機能で、変更箇所が明確に追えます。

03 / 専用エディタ

編集はセルで。整合性はスキーマで。

マスターデータを Excel で組んで CSV インポート、参照整合性は VLOOKUP で目視確認 ── この回りくどさを終わらせるための専用エディタ。スプレッドシートの操作感はそのまま、整合性は型システムが保証します。

  • スプレッドシート感覚のセル操作

    セル単位の選択、Shift / Ctrl による拡張選択、矩形範囲のドラッグ、ダブルクリックでインライン編集。表計算と同じ手の動きでデータを組めます。

  • 完全なデータバリデーション

    スキーマが各カラムの型と制約を保証。VLOOKUP で参照整合性を確認する必要はもうありません。保存時 / Cook 時に自動検証。

04 / インデックス

属性による、高速な逆引き。

  • インデックス定義

    スキーマに任意の属性をインデックスとして宣言し、カテゴリ・レアリティ・ファクションなど、ドメイン特有のキーで Row をグループ化できます。

  • セカンダリクエリ

    Blueprint・C++ どちらからも、定義したインデックス経由で O(1) に近い効率で Row を検索。行数が増えても性能が劣化しません。

01 / Maintainability

Renames don't break references.

DataTable row references are FName strings — rename a row and every reference silently breaks. DataIndexer assigns a GUID (FDataIndexerPrimaryKey) to every row, so references survive any rename.

  • GUID-based PrimaryKey

    Change a key's name freely — Blueprint and code references stay intact.

  • Key usage tracking

    See which assets and code reference a given row, directly in the editor.

02 / Composition

Layer data with inheritance.

  • Repository inheritance and merging

    Covers the same ground as UCompositeDataTable — inherit from a parent repository or merge multiple repositories, with type-safe schemas throughout.

  • Per-row overrides

    Override only specific rows in a child repository — a capability UCompositeDataTable didn't have. Changes stay traceable and clear.

03 / Dedicated Editor

Edit in cells. Schema enforces integrity.

Building master data in Excel, importing via CSV, then verifying referential integrity by hand with VLOOKUP — that roundabout workflow ends here. A dedicated editor with spreadsheet-feel editing and type-system-backed consistency.

  • Spreadsheet-style cell editing

    Per-cell selection, Shift / Ctrl range extension, rectangular drag selection, and double-click inline editing. The same hand motions as a spreadsheet, inside the UE editor.

  • Full data validation

    The schema enforces types and constraints on every column. No more manual VLOOKUP checks for referential integrity. Validation runs automatically on save and cook.

04 / Indexes

Fast lookup by any attribute.

  • Index definition

    Declare any row attribute as an index on the schema — category, rarity, faction. Domain-specific keys group rows naturally.

  • Secondary queries

    Look up rows by an index from Blueprint or C++ at near-O(1) efficiency. Row count growth no longer degrades query performance.

Core Concepts

Understand these four pieces and everything clicks.

Repository FDataIndexerPrimaryKey

The data asset that holds rows. Each row is identified by a FDataIndexerPrimaryKey (GUID) and stores a typed struct via FInstancedStruct. Repositories can reference parent repositories for hierarchical data composition.

Learn more →
Schema UDataIndexerSchema

Defines the row struct and editor behavior. A schema controls display names, property text rendering, custom widgets, and the index key builders that power secondary lookups.

Learn more →
Keys & Handles FDataIndexerRowHandle

FDataIndexerPrimaryKey uniquely identifies a single row. FDataIndexerRowHandle pairs a repository with a key for Blueprint-friendly references.

Learn more →
Indexes FDataIndexerIndexKey

Secondary lookup dimensions defined on a schema. Each index maps rows to a FDataIndexerIndexKey, enabling fast lookup by any domain-specific attribute.

Learn more →
