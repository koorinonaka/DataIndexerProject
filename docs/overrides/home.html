01 / 参照安全性

リネームしても壊れない。

// Secondary-index lookup — O(1) regardless of row count
FItemRow Query;
Query.Rarity = EItemRarity::Rare;

TArray<FDataIndexerPrimaryKey> RareKeys =
    FItemInterface::GetPrimaryKeys(
        *ItemRepo, UItemSchema::ByRarityIndex(), Query
    );

for (const FDataIndexerPrimaryKey& Key : RareKeys)
{
    const FItemRow* Row = FItemInterface::FindRow(*ItemRepo, Key);
    if (Row) { /* ... */ }
}

コアコンセプト

Repository

データを格納するアセット。各 Row は FDataIndexerPrimaryKey（GUID）で識別され、スキーマが定義した型の構造体を保持します。親リポジトリを参照することで階層的なコンポジションが行えます。

→ リポジトリ Schema

Row の構造体とエディタ挙動を定義します。表示名、プロパティのレンダリング、カスタムウィジェット、インデックスキーのビルダーを制御します。

→ スキーマ Keys & Handles

FDataIndexerPrimaryKey は単一の Row を一意に識別します。FDataIndexerRowHandle はリポジトリとキーをペアにして Blueprint で扱いやすくしたものです。

→ キーとハンドル Indexes

スキーマに定義するセカンダリ検索軸。各インデックスは Row を FDataIndexerIndexKey にマッピングし、任意の属性による高速検索を実現します。

Core Concepts

Repository

The data asset that holds rows. Each row is identified by a FDataIndexerPrimaryKey (GUID) and stores a typed struct via FInstancedStruct. Repositories can reference parent repositories for hierarchical data composition.

→ Repository Schema

Defines the row struct and editor behavior. A schema controls display names, property text rendering, custom widgets, and the index key builders that power secondary lookups.

→ Schema Keys & Handles

FDataIndexerPrimaryKey uniquely identifies a single row. FDataIndexerRowHandle pairs a repository with a key for Blueprint-friendly references.

→ Keys & Handles Indexes

Secondary lookup dimensions defined on a schema. Each index maps rows to a FDataIndexerIndexKey, enabling fast lookup by any domain-specific attribute.

→ Indexes
