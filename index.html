    UNREAL ENGINE PLUGIN

    References that
    survive renames.

    DataIndexer is a game data management plugin for UE5. Every row is identified by a GUID (FDataIndexerPrimaryKey), so references survive renames, moves, and reimports. A type-safe C++ / Blueprint API and a centralized Schema keep maintenance costs low as your data grows.

    What DataIndexer Adds

    Spreadsheet-feel editing to type-safe management.

    01 / Maintainability

    Renames don't break references.

    DataTable row references are FName strings — rename a row and every reference silently breaks. DataIndexer assigns a GUID (FDataIndexerPrimaryKey) to every row, so references survive any rename.

    • GUID-based PrimaryKey

      Change a key's name freely — Blueprint and code references stay intact.

    • Key usage tracking

      See which assets and code reference a given row, directly in the editor.

    02 / Composition

    Layer data with inheritance.

    • Repository inheritance and merging

      Covers the same ground as UCompositeDataTable — inherit from a parent repository or merge multiple repositories, with type-safe schemas throughout.

    • Per-row overrides

      Override only specific rows in a child repository — a capability UCompositeDataTable didn't have. Changes stay traceable and clear.

    03 / Dedicated Editor

    Edit in cells. Schema enforces integrity.

    Building master data in Excel, importing via CSV, then verifying referential integrity by hand with VLOOKUP — that roundabout workflow ends here. A dedicated editor with spreadsheet-feel editing and type-system-backed consistency.

    • Spreadsheet-style cell editing

      Per-cell selection, Shift / Ctrl range extension, rectangular drag selection, and double-click inline editing. The same hand motions as a spreadsheet, inside the UE editor.

    • Full data validation

      The schema enforces types and constraints on every column. No more manual VLOOKUP checks for referential integrity. Validation runs automatically on save and cook.

    04 / Indexes

    Fast lookup by any attribute.

    • Index definition

      Declare any row attribute as an index on the schema — category, rarity, faction. Domain-specific keys group rows naturally.

    • Secondary queries

      Look up rows by an index from Blueprint or C++ at near-O(1) efficiency. Row count growth no longer degrades query performance.

    In Practice

    The same data. C++ or Blueprint.

    C++ queries via a type-safe native interface. Blueprint queries via Get Row nodes. Both read from the same Repository.

    C++ and Blueprint share the same Repository 
    // 行構造体と Interface エイリアスを定義
USTRUCT(BlueprintType)
struct FItemRow
{
    GENERATED_BODY()

    UPROPERTY(EditAnywhere, BlueprintReadWrite) FText       DisplayName;
    UPROPERTY(EditAnywhere, BlueprintReadWrite) EItemType   Type;
    UPROPERTY(EditAnywhere, BlueprintReadWrite) EItemRarity Rarity;
    UPROPERTY(EditAnywhere, BlueprintReadWrite) int32       BaseValue = 0;
};
using FItemInterface = DataIndexer::TNativeSchemaInterface<FItemRow>;

// 特定の行を取得
if (const FItemRow* Row = FItemInterface::FindRow(Repository, PrimaryKey))
{
    FText Name = Row->DisplayName;
}

// Index で属性別に逆引き
FItemRow Filter; Filter.Rarity = EItemRarity::Rare;
for (const FDataIndexerPrimaryKey& K :
     FItemInterface::GetPrimaryKeys(
         Repository, UItemSchema::ByRarityIndex(), Filter))
{
    Process(*FItemInterface::FindRow(Repository, K));
}
